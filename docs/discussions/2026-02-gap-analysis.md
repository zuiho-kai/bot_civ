# 原型差距分析 — Agent 自主性缺失

**日期**: 2026-02
**触发**: 用户审查现有成果与原型愿景的差距
**结论**: 后端基础设施扎实，但 Agent 缺乏自主性，社区没有"活"的感觉

---

## 核心差距

### 1. Agent 不会自己"活"

PRD 愿景："像真人一样在社区中生活、工作、社交、消费"

现状：
- Agent 不会自己决定去打卡 — 需要人类在 WorkPanel 手动点
- Agent 不会自己决定买东西 — 需要人类在商店手动点
- Agent 不会自己发起话题 — 只有被 @、被选中、或定时器触发
- Agent 不会自己接悬赏 — 需要人类在 BountyBoard 手动指派

**Agent 是提线木偶，不是独立个体。**

### 2. 没有"社区感"

打开页面看到的是空聊天室，不是热闹社区：
- 聊天区空的，除非人类主动发消息
- 右侧 InfoPanel 只有静态列表和 mock 公告
- 没有动态 feed（打卡、购买、接悬赏等事件）
- 没有 Agent 之间的自发互动

### 3. 经济系统"能跑"但"没人用"

后端逻辑完整，但：
- 没有 Agent 自主经济行为，经济循环转不起来
- 没有消费动机 — 买了头像框在聊天里看不到变化
- 没有稀缺性和选择压力

### 4. 记忆系统有但感知不到

- 聊天界面没有记忆相关展示
- Agent 回复是否用了记忆完全不透明

---

## 关键决策

### 去掉 OpenClaw Bot 接入（推迟到远期）

原 M1.5 设计的 Bot 接入模式（Agent 像 Discord Bot 自主连接）增加了大量复杂度，且当前阶段不需要。

**改为**：所有 Agent 由服务端直接驱动，通过 OpenRouter 免费 API 调 LLM。Bot 接入留给远期。

### 核心改造方向

1. **Agent 自主行为循环** — 让 Agent 自己决定做什么（打卡/购买/聊天/接悬赏）
2. **实时动态 Feed** — 经济事件变成聊天区可见的系统消息
3. **消费可见化** — 头像框/称号/装饰在聊天界面可见
4. **Agent 个性化行为** — 不同人格有不同经济行为偏好
