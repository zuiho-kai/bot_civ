# 任务看板方案调研（开发管理用）

> 日期：2026-02-18
> 状态：遗留问题，需求不明确，方案未定
> 用途：多 Claude Code agent 并行开发时的任务协调

## 背景

项目开发中可能需要多个 Claude Code agent 并行工作，需要一个任务看板来协调分工、避免冲突。

## 方案对比

| 维度 | 文件 + 锁 | SQLite + WAL | 本地 API | 外部服务 |
|------|-----------|-------------|---------|---------|
| 并发安全 | 差（Windows 无 flock） | 良好（原子 SQL） | 最好 | 最好 |
| 实现复杂度 | 低但易出错 | 中低 | 高 | 中等 |
| 离线可用 | 完全 | 完全 | 完全 | 无 |
| CLI 友好度 | 读好/写差 | 很好 | 中等 | 差 |
| Windows 兼容 | 差 | 好 | 好 | 好 |

## 初步倾向

SQLite + WAL 并发安全最好，但用户反馈：手动改不方便（得写 SQL），且需求还不明确。

## 待明确的问题

1. 具体使用场景是什么？（多 agent 分工？进度追踪？冲突避免？）
2. 人类需要多频繁手动编辑任务？（如果频繁，JSON/YAML 可读性更重要）
3. 并发写入频率有多高？（低频的话文件方案也够用）
4. 是否需要和现有工具（GitHub Issues 等）集成？
